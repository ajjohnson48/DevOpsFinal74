SECTION 2: NEW FEATURE VERIFICATION
--------------------------------------------------------------

Random Password Generation:
  STATUS: VERIFIED
  EVIDENCE: generate_password() reads /dev/urandom, filters to A-Za-z0-9
  via tr -dc, head -c 12. Tests 9a/9b/9c verify length (12), charset
  (alphanumeric only), and uniqueness across consecutive calls.

Credentials File Hardening:
  STATUS: VERIFIED
  EVIDENCE: write_credential() sets umask 077, install -d -m 700 for the
  directory, install -m 600 -o root -g root for the file. Tests 9d/9e
  confirm the on-disk mode (600) and owner (root:root). Same protection
  model as /etc/shadow.

Forced Password Change at First Login (chage -d 0):
  STATUS: VERIFIED
  EVIDENCE: Both create_user() and rotate_all_passwords() call
  `chage -d 0 "$username"` after chpasswd succeeds. Test 9f runs
  `chage -l alpha.one` and confirms the "Password must be changed"
  marker is present.

Password Rotation (Menu Option 4):
  STATUS: VERIFIED
  EVIDENCE: rotate_all_passwords() reads usernames from
  CREATED_USERS_LOG (never /etc/passwd), confirms with y/n, generates
  new random password per user, runs chpasswd + chage -d 0, appends a
  "(rotated)" tagged row to the credentials file. Tests 9g/9h/9i verify
  the rotation summary, the missing-user skip path, and the append
  semantics. Test 9j confirms cryptographically that the old
  deterministic password is no longer the active one.

Source-of-truth invariant:
  STATUS: VERIFIED
  EVIDENCE: Rotation reads only from CREATED_USERS_LOG. Manual
  inspection confirms `root`, `nobody`, `systemd-resolve`, `ubuntu`,
  and other system accounts are never touched.
