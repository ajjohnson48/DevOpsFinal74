Bug #2/#7 - Scraper output pollution (status messages going to stdout):
  STATUS: FIXED
  EVIDENCE: scrape_faculty() sends all status messages to stderr (>&2 redirects
  on lines 193, 198-199, 215-216). The stdout name list contains exactly 42
  clean entries with no box-drawing characters, status messages, or garbage.

Bug #3 - Spaces in names producing invalid usernames:
  STATUS: FIXED
  EVIDENCE: Input "Mary Jane" as first name is rejected with "First name
  contains invalid characters. Only letters, hyphens, and apostrophes are
  allowed." The regex ^[a-zA-Z\'-]+$ on line 431 blocks spaces.

Bug #4 - Long usernames exceeding 32-char limit:
  STATUS: FIXED
  EVIDENCE: 50-character first+last names are accepted by input validation
  (they contain only letters) but create_user() catches the >32 char username
  at line 309-310 and shows "exceeds 32-character limit." No system user created.

Bug #5 - Shell metacharacters accepted in names:
  STATUS: FIXED
  EVIDENCE: "test;whoami", "test|cat", "${HOME}", and "$(whoami)" are all
  rejected by the regex whitelist ^[a-zA-Z\'-]+$ with "invalid characters"
  error. Both manual_add_user() (lines 431-438) and create_user() (line 299)
  enforce this validation.

Bug #6 - EOF causing infinite loop when piping input:
  STATUS: FIXED
  EVIDENCE: Piping empty input causes script to print "End of input detected.
  Exiting." and exit with code 0. The fix is `read -r choice || { ... exit 0; }`
  on line 525. No timeout needed; script exits immediately.

Bug #8 - No signal trap for clean interrupt:
  STATUS: FIXED
  EVIDENCE: Line 519 has: trap 'echo ""; print_info "Interrupted. Exiting.";
  exit 130' INT TERM. Functional test confirms script exits on SIGINT signal.
